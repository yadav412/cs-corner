API's lesson 1

API- Application Programming Interfaces

Set of rules used to intergrated external software services(payment,maps) without building their own services from scratch.

Endpoints/URI's - URI's- Uniform resource identifier - unique identifers for resources.

Example

fetch(URL)

Main types of API's

REST- Representational State Transfer - uses https to fetch data from a external server 

Client(user/coder)-> Server: request/interacts with data -> Client : confirms on action on data

HTTP methods

"GET"- retrives data from a collection of data (read)


        fetch('https://api.example.com/data')

"POST" - creates a new resource within the collection (write)


        fetch('https://api.example.com/data',{ 

           method : 'POST', 

           body: JSON.stringify({name:'Yadav'})
        })

"PUT" - updates an existing resource (complete overhaul)


        fetch('https://api.example.com/data',{

            method: 'PUT',

            body: JSON.stringify ({name:'Yadav Updated',age :21})
        })

"PATCH" - performs a partial change - "patch work change" (update)


        fetch('https://api.example.com/data',{

            method:'PATCH',

            body: JSON.stringify ({age :22})
    })

"Delete" - the 'nuke' deletes a resource


        fetch('https://api.example.com/data',{

            method:'DELETE', 

        })
